Depuració del codi

L’objectiu d’aquesta activitat és dominar el concepte de punt d’interrupció (o breakpoint) i la seva aplicació. Es demana la col·locació de punts d’interrupció per poder anar executant, pas a pas, el codi, validant els valors que van adquirint les variables.

S’ha de crear un nou projecte de Java que permeti calcular el factorial. S’ofereix, a continuació, el codi de programació que soluciona el factorial. 

  1. public class Factorial {
  2.  
  3. 	public static double factorial (double numero) {
  4.  
  5. 		if (numero==0)
  6. 			return 1;
  7. 		else
  8. 		{
  9. 			double resultat = numero * factorial(numero-1);
  10. 			return resultat;
  11. 		}
  12. 	}	
  13.  
  14. 	public static void main(String[] args) {
  15.  
  16. 		System.out.println(factorial(5));
  17.  
  18. 	}
  19.  
  20. }

Es demana depurar el codi pas a pas, introduint els punts d’interrupció necessaris i mostrant el valor que van tenint les variables.

A continuació, a les següents figures, es mostrarà, pas a pas, com solucionar l’exercici plantejat.

A partir del projecte creat amb el codi indicat a l’enunciat, s’indica com començar la seva depuració.

A continuació es mostren les interfícies que veu l’usuari en seleccionar la funcionalitat de depuració del menú. Es podrà seleccionar el codi que es vol depurar i especificar les condicions que es vulguin. Concretament, es determina un punt d’interrupció a l’hora de retornar el valor calculat a la funció factorial.

A continuació es mostra un detall dels valors de les variables numero i resultat en un moment determinat de l’execució del codi de programació. En concret, a la figura 3 es mostra l’instant en què es passa per segona vegada per la línia de codi de programació amb el punt d’interrupció.

A continuació es mostra el valor de les variables numero i resultat la tercera vegada que es passa pel punt d’interrupció.

A continuació es mostra el valor de les variables numero i resultat la quarta vegada que es passa pel punt d’interrupció. En aquest cas, el valor de numero és 4 (en ser la quarta vegada que s’ha finalitzat la recursivitat de la crida a la funció factorial) i el valor temporal de resultat és 24.

JUnit. Prova unitària

L’objectiu d’aquesta activitat és aprendre a crear proves unitàries amb JUnit.

Es demana:

  1. Crear un projecte que implementi una pila.
  2. Codificar les proves unitàries amb JUnit, que permetin validar el correcte funcionament de la pila.

Una pila és una llista o estructura de dades en què la forma d’accés als seus elements és de tipus LIFO (de l’anglès Last In First Out, el darrer a entrar és el primer a sortir), que permet emmagatzemar i recuperar dades.

Recordar que JUnit és un framework per automatitzar les proves unitàries d’aplicacions Java; permet l’execució de classes Java de manera controlada, amb l’objectiu de poder avaluar si el funcionament de cada un dels mètodes de la classe es comporta com s’espera.

1. Crear un projecte que implementi una pila: 

  1. import java.util.Vector;
  2.  
  3. public class Pila {
  4.  
  5. 	private int mida;    
  6. 	private Vector<Integer> pila;
  7.  
  8.     public Pila() {
  9.          super();
  10.          pila = new Vector<Integer>();
  11.          mida = 0;
  12.      }
  13.  
  14.     public boolean pilaBuida () {
  15.          if (mida==0) {
  16.              return true;
  17.          }
  18.          return false;
  19.      }
  20.  
  21.     public void apilar ( Integer o ) {
  22.          pila.add(mida, o);
  23.          mida++;
  24.      }
  25.  
  26.     public Integer desapilar () {
  27.  
  28.         try {
  29.              if(pilaBuida())
  30.                  throw new ErrorPilaVacia();
  31.              else {
  32.                  return pila.get(--mida);
  33.              }
  34.          } catch(ErrorPilaVacia error) {
  35.              System.out.println("ERROR: la pila està buida");
  36.              return null;
  37.          }
  38.      }
  39.  
  40.     public int getNumElements() {
  41.          return mida;
  42.      }
  43.  
  44.     @SuppressWarnings("serial")
  45.     class ErrorPilaVacia extends Exception {
  46.         public ErrorPilaVacia() {
  47.             super();
  48.         }
  49.     }
  50. }

2. Codificar les proves unitàries amb JUnit, que permetin validar el correcte funcionament de la pila:

En primer lloc, cal descarregar una versió de JUnit.

A partir del projecte Pila es podrà incloure una nova classe de test que anomenarem PilaTest.

La classe de proves s’ha de crear en el mateix paquet on es troba la nostra classe. Això ens permetrà tenir accés a les variables declarades sense identificador d’accés, sense necessitat d’un mètode altern com un setter.

A continuació es mostra com començar la utilització de JUnit creant un JUnit Test Case.

A continuació es pot observar el diàleg que s’haurà de configurar per crear les proves, indicant la carpeta amb què es treballarà, el nom de les proves i la classe que se sotmetrà a les proves.

A continuació es podrà seleccionar quins mètodes de la Classe Pila es voldran verificar.

Automàticament s’haurà inclòs en el projecte la referència a la llibreria.

En el cas que s’hagi d’especificar manualment la referència, caldrà que s’afegeixi la llibreria de JUnit al Build Path. Caldrà clicar amb el botó dret a sobre del projecte i seleccionar sobre el menú contextual l’opció Properties. Es pot observar a continuació:

Taula de continguts: Ofereix una vista general de l'estructura del mòdul, que us facilitarà la navegació a través dels seus continguts.
Opcions de format: Permet configurar el format de lectura a partir de les vostres preferències, modificant la mida de la lletra, el color del fons, l'amplada de la pàgina o l'ocultació dels continguts complementaris, entre d'altres.
Imprimir: Envia el contingut seleccionat a la impressora.
Prefereits: Permet desar aquelles parts del contingut a les que us interessi accedir en un moment posterior; us permetrà anar creant un repositori personalitzat de les seccions que considereu més rellevants. Un cop l'hageu començat a fer servir se us mostrarà a sota un comptador que us informarà del nombre de preferits que s'hagin emmagatzemat fins al moment.
Comptador i llistat de preferits: Indica el nombre de preferits que s'han desat fins el moment; clicant damunt seu accedireu al llistat de preferits, i a través d'aquest podreu accedir directament als continguts que hageu emmagatzemat.
Ajuda: Aquesta opció activa la informació de les funcionalitats del web tot situant el punter del ratolí al damunt dels diferents ítems.
Mostrar/Ocultar barra lateral: Deshabilita la barra d'opcions, permetent la seva recuperació quan es desitgi.
Cerca: Introduïu les paraules clau sobre aquells conceptes que desitgeu localitzar en els continguts del mòdul. També podeu excloure un terme de la cerca tot afegint-hi el signe “-” al davant.
Capçalera: Indica l'apartat o secció que es mostra en pantalla. Tingueu present que les capçaleres de segon, tercer i quart nivell també es poden desar com a marcadors.
